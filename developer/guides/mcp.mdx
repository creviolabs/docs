The Crevio MCP server lets AI agents — like Claude, Cursor, or custom LLM workflows — manage your store through natural language. It exposes two tools: one to discover available API endpoints, and one to execute them. Two tools, full API coverage, minimal context usage.

## What is MCP?

[Model Context Protocol (MCP)](https://modelcontextprotocol.io) is an open standard that gives AI assistants a way to connect to external tools and data sources. Instead of manually writing API calls, an AI agent discovers and invokes your Crevio API through MCP — using natural language.

## How it works

Traditional MCP servers expose one tool per API endpoint. This works for small APIs, but quickly becomes impractical — each tool definition consumes tokens in the AI model's context window.

Crevio takes a different approach with just two tools:

1. **`search`** — The agent queries the OpenAPI spec with keywords to find relevant endpoints and their parameters
2. **`execute`** — The agent calls those endpoints by specifying an HTTP method, path, and optional parameters

The agent never sees the full API spec. It discovers only what it needs, keeps context lean, and gets the same responses as the [REST API](/developer/guides/api-overview).

## Base URL

```
https://mcp.crevio.co/mcp
```

The server uses **Streamable HTTP** transport — a single `POST` endpoint that accepts JSON-RPC requests and returns JSON-RPC responses.

## Authentication

All MCP requests require a **Bearer token** in the `Authorization` header. Use the same API tokens from your [Developer settings](/developer/guides/api-overview#creating-api-tokens).

```bash
curl -X POST https://mcp.crevio.co/mcp \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
```

<Warning>
  MCP tokens have the same permissions as API tokens. Only share them with trusted AI tools and agents.
</Warning>

## The two tools

### `search` — Discover API endpoints

A **read-only, idempotent** tool for exploring the Crevio API. Pass a keyword query to find matching endpoints:

```json
{
  "name": "search",
  "arguments": { "query": "products" }
}
```

Returns matching endpoints with method, path, summary, parameters, and request body schemas:

```json
[
  { "method": "GET", "path": "/products", "summary": "List products", ... },
  { "method": "POST", "path": "/products", "summary": "Create product", ... },
  { "method": "GET", "path": "/products/{prefix_id}", "summary": "Get product details", ... }
]
```

Pass an empty query to list all 59 available endpoints.

### `execute` — Run API operations

Executes an authenticated API call against your account. Specify an HTTP method, path, and optional parameters:

```json
{
  "name": "execute",
  "arguments": {
    "method": "get",
    "path": "/products"
  }
}
```

Paths are automatically prefixed with `/v1`. Parameters are sent as query params for `GET`/`DELETE` and as a JSON body for `POST`/`PATCH`:

```json
{
  "name": "execute",
  "arguments": {
    "method": "post",
    "path": "/products",
    "params": {
      "product": { "name": "My Course", "product_type": "course" }
    }
  }
}
```

More examples:

```json
// Update a product
{ "method": "patch", "path": "/products/prod_abc123", "params": { "product": { "name": "Updated Name" } } }

// Delete a product
{ "method": "delete", "path": "/products/prod_abc123" }

// List orders
{ "method": "get", "path": "/orders" }

// Create a discount
{ "method": "post", "path": "/discounts", "params": { "discount": { "code": "SUMMER20", "discount_type": "percent_off", "percent_off": 20 } } }
```

The `execute` tool dispatches requests through the **same API controllers** as `api.crevio.co` — all authorization, validation, and rate limiting apply. Responses match the [REST API format](/developer/guides/api-overview#conventions) exactly.

## Connecting to Claude Desktop

Add this to your Claude Desktop configuration file (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "crevio": {
      "url": "https://mcp.crevio.co/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_TOKEN"
      }
    }
  }
}
```

After restarting Claude Desktop, you can ask Claude to manage your store in plain English:

> "List all my products and their prices"
>
> "Create a 20% discount code called SUMMER20 that expires next month"
>
> "Who are my top 5 customers by order count?"

## Connecting to Cursor

In Cursor, go to **Settings** > **MCP Servers** and add a new server:

- **Name**: Crevio
- **Type**: Streamable HTTP
- **URL**: `https://mcp.crevio.co/mcp`
- **Headers**: `Authorization: Bearer YOUR_API_TOKEN`

## Other MCP clients

Any MCP-compatible client that supports Streamable HTTP transport can connect. The server implements the standard [MCP specification](https://modelcontextprotocol.io/specification) with JSON-RPC over HTTP.

## Example: How an agent handles a request

When you ask *"Create a buy-one-get-one discount for my course"*, here's what happens under the hood:

<Steps>
  <Step title="Discover relevant endpoints">
    The agent calls `search` with `{"query": "discount"}` to find the discount creation endpoint and its required parameters.
  </Step>
  <Step title="Find the product">
    The agent calls `execute` with `{"method": "get", "path": "/products"}` to list products and identify the course.
  </Step>
  <Step title="Create the discount">
    The agent calls `execute` to create the discount:
    ```json
    {
      "method": "post",
      "path": "/discounts",
      "params": {
        "discount": {
          "code": "BOGO",
          "discount_type": "percent_off",
          "percent_off": 100,
          "duration": "once",
          "applies_to_product_ids": ["prod_abc123"]
        }
      }
    }
    ```
  </Step>
  <Step title="Confirm the result">
    The agent formats the response and tells you the discount was created with code `BOGO`.
  </Step>
</Steps>

The whole interaction uses just 2-3 tool calls regardless of how many API endpoints Crevio has.

## What's available

Everything accessible through the [REST API](/developer/guides/api-overview#what-you-can-do-with-the-api) is available via MCP:

| Category | Operations |
|----------|-----------|
| **Account** | Retrieve account details |
| **Products** | Create, update, delete, list, retrieve |
| **Price variants** | Create, update, list, retrieve |
| **Orders** | List, retrieve |
| **Customers** | List, create, update, retrieve |
| **Subscriptions** | List, retrieve, cancel, pause, resume |
| **Checkouts** | Create sessions, retrieve, update |
| **Carts** | Create, retrieve, update, delete |
| **Discounts** | Create, update, delete, list, retrieve |
| **Checkout links** | Create, update, delete, list, retrieve |
| **Experiences** | List, retrieve |
| **Blog posts** | List, retrieve |
| **Invoices** | Create, list, retrieve, void, pay |
| **Webhooks** | Create, update, delete, list, retrieve, test |
| **Reviews** | List, retrieve |
| **x402** | Get and update configuration |

## Security

All operations run within your account's tenant boundary — the same multi-tenant isolation as the REST API. The MCP server dispatches requests through the same API controllers as `api.crevio.co`. No special code paths — the agent gets the same behavior, validation, and error messages as direct API calls.
